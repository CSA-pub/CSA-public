# Security Policy

## Reporting a vulnerability or data concern

We take the integrity of the CSA data seriously. If you discover any of the
following, please report it privately:

- A security vulnerability in the site ([csa-public.pages.dev](https://csa-public.pages.dev))
  or in anything published in this repository (e.g. the starter notebook).
- A data concern — including an **incorrect asset↔ticker link** that could mislead
  someone about which security an asset maps to.
- Any suspected leak of the full (paid) dataset or pipeline source.

**Please email: roasterdb@proton.me**

Do **not** open a public GitHub issue for security-sensitive reports.

## What to include

- A clear description of the issue and its impact.
- Steps to reproduce (for site/notebook issues) or the affected `asset` / `ticker`
  / `nct_id` (for data concerns).
- Your contact details if you'd like a response.

## Our commitment

- We aim to acknowledge reports within **5 business days**.
- We will keep you informed as we investigate and resolve the issue.
- We will not pursue legal action against good-faith security research that
  respects user privacy and avoids service disruption or data destruction.

## Scope

This repository contains only a **public sample** and documentation — it holds no
credentials, pipeline source, or full dataset. Reports about those (e.g. a
suspected leak) are still very welcome via the email above.

## Not investment advice

CSA is a dataset, not financial or investment advice. Estimated catalyst dates
routinely slip; verify anything material against each row's `source_url`.
